# Coding Standards

## General

- Prefer simple, readable code over clever code
- Keep functions small and focused
- Use clear names for files, classes, functions, and variables
- Avoid large unreviewed changes
- Update documentation when behavior changes

## Git

- Use feature branches
- Do not commit directly to `main` unless necessary
- Use short imperative commit messages

Examples:

```txt
Update README for Milestone 1
Add Unity player movement prototype
Add backend health endpoint
Connect Unity client to backend
